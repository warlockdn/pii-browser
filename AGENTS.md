# PII Browser Project Agent Guide

## Project Snapshot
- Stack: Bun + Next.js App Router + TypeScript + Tailwind v4 + shadcn/ui.
- Inference: browser-only via `@huggingface/transformers`.
- Model: `onnx-community/bert-small-pii-detection-ONNX`.
- No server-side text processing for detection/redaction.

## Core Files
- `app/page.tsx`: main demo UI, single-text detection, 20-sample benchmark runner, status/timing badges.
- `lib/pii.ts`: inference singleton cache, entity normalization, span resolution fallback, redaction builder.
- `app/layout.tsx`: app metadata + `Toaster` mount.
- `components/ui/*`: shadcn components generated by CLI.

## Product Behavior (Must Keep)
- First detection lazily downloads model from Hugging Face CDN.
- Later detections reuse cached pipeline instance in browser session.
- Detection uses token-classification with aggregation (`simple`) to emit entity spans/entities.
- Redaction replaces detected entities with label placeholders, preserves non-PII text.
- Batch benchmark runs 20 built-in samples sequentially and shows per-sample latency.
- Benchmark status colors:
  - Green => PII found.
  - Red => no PII found.

## Guardrails
- Keep inference client-only unless explicitly asked to add server/hybrid mode.
- Do not send input text to any backend/API in default flow.
- Preserve current state model in `app/page.tsx`:
  - `idle`, `model-loading`, `detecting`, `success`, `empty-result`, `error`.
- Avoid replacing shadcn-generated UI with custom primitives unless necessary.

## Development Commands (Bun)
- Install deps: `bun install`
- Dev: `bun run dev`
- Lint: `bun run lint`
- Build: `bun run build`

## Debug Notes
- Model output may omit `start/end`; `lib/pii.ts` includes word-based fallback span matching.
- Aggregated model words may contain spaced punctuation (`"test @ example. com"`); normalization logic must be preserved when editing parser.

## When Updating Samples/Benchmark
- Keep `BENCHMARK_SAMPLES` count at 20 unless user requests otherwise.
- Keep each row metrics: `ms`, `entityCount`, `foundPii`.
- Keep summary metrics: total ms, average ms, found count, not-found count.
